<audio> HTML5 introdujo el elemento <audio> para incluir archivos de audio en sus páginas. Al igual que con el video HTML5, los navegadores esperan diferentes formatos para el audio. El elemento <audio> tiene un número de atributos que le permiten controlar la reproducción de audio:

src:este atributo especifica la ruta de acceso al archivo de audio.

controls:este atributo indica si el reproductor debe mostrar los controles. Si no utiliza este atributo, no se mostrarán controles de forma predeterminada. También puede especificar sus propios controles mediante JavaScript.

autoplay: la presencia de este atributo indica que el audio debe empezar a reproducirse automáticamente. (Se considera una mejor práctica permitir que los visitantes elijan reproducir el audio).

loop: este atributo espesifica que la pista de audio debe reproducirse de nuevo una vez que haya finalizado






src: el elemento <source> utiliza el atributo src para indicar dónde se encuentra el archivo de audio. tipo en el momento de escribir, el atributo de tipo no se usaba habitualmente en el elemento <source> de la misma manera que lo era para elemento <video>.











Youtube video ID Youtube mostrará un ID (como EF81Pb5IWoU), al guardar (o reproducir) un video. Puede utilizar este ID y hacer referencia a su vídeo en el código HTML.

Reproducción de un video de Youtube en HTML:
Para reproducir el vídeo en una página web, haga lo siguiente:

  • Sube el video a Youtube
  • Anote el ID del video
  • Defina un elemento <iframe> en su página web
  • Deje que el atributo src apunte a la URL del vídeo
  • Utilice los atributos width y height para especificar la dimensión del reproductor

En HTML5 no es necesario proporcionar valores para todos los atributos, como los controles, reproducción automatica y atributos de bucle utilizados con el elemento <video>. Estos atributos son como interruptores de encendido/apagado. Si el atributo está presente, activa esta opción. Si se omite el atributo, la opción está desactivada. Si el navegador no admite el elemento <video> o el formato de vídeo utilizado, se mostrará lo que esté entre la <video> de apertura y las etiquetas de </video> de cierre.

preload: este atributo le indica al navegador qué hacer cuando se carga la página. Puede tener uno de tres valores:
none: El navegador no debe cargar el video hasta que el usuario presiona play.
auto: El navegador debe descargar el video cuando se carga la página.
metadata: El navegador debe simplemente recopilar información como el tamaño, el primer fotograma, la lista de pistas y la duración.

La etiqueta video: Para especificar la ubicación del archivo que se reproducirá, puede usar el elemeto <source> dentro del elementro <video>. (Esto debe reemplazar el atributo src en la etiqueta <video> de apertura) También puede utilizar varios elementos <source> para especificar que el video está disponible em diferentes formatos. (Debido a un error en el iPad, debe proporcionar el vídeo MP4 como el primer formato. De lo contrario, puede que no inicie.)

src: Este atributo especifica la ruta de acceso al video.

type:Se debe utilizar este atributo para decirle al navegador qué formato es el vídeo. De lo contrario, descargará parte del video para ver si puede reproducir el archivo (que tomará tiempo y ancho de banda).

codecs: el códec que usó para codificar el vídeo se suministra dentro del atributo type. Tenga en cuenta el uso de comillas simples, así como comillas dobles en el atributo de tipo, cuando se suministra.
